⚠️ Critical DoorsOS Update Warning

8 June 2025

A new experimental update for DoorsOS has been released that introduces advanced thread management features using a new CPU thread handler. However, this update may cause some devices to **fail to boot** if the CPU does not support the required instruction set or thread allocation.

If your device fails to boot after this update, **do not panic**. A recovery method is built-in. Here's how to fix it:

Recovery Instructions:

  1. Hold F12 while powering on to boot into the DoorsOS Recovery Console.
  2. Once loaded, type the following custom recovery commands in order:
set threadsafe-mode 0
fix.bootconfig --safe
reinit.cpu
  1. After entering these commands, type:
sys.reboot

If successful, your device will emit a **short single beep** after rebooting. If you hear nothing or the device returns to recovery, re-check your commands and try again.

Thank you for using DoorsOS. As always, backup your system before major updates.
